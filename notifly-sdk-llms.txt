# Platform: iOS

## 코어
- [Notifly 메인 엔트리](https://raw.githubusercontent.com/team-michael/notifly-ios-sdk/refs/heads/main/Sources/Notifly/notifly-ios-sdk/notifly-ios-sdk/SourceCodes/Notifly/Notifly.swift): Notifly iOS SDK의 싱글톤 엔트리 포인트. 초기화 후 Tracking/User/Notifications/InApp 매니저를 보유하며, UserDefaults 네임스페이스 등록 및 프로젝트/계정 자격 증명 보관을 수행.
- [퍼블릭 API](https://raw.githubusercontent.com/team-michael/notifly-ios-sdk/refs/heads/main/Sources/Notifly/notifly-ios-sdk/notifly-ios-sdk/SourceCodes/Notifly/Notifly+PublicAPI.swift): 공개 인터페이스. FirebaseApp 초기화 전제, 프로젝트/계정 설정으로 initialize 수행, APNs/FCM 토큰 연계, 세션 시작(trackSessionStartInternalEvent), 포그라운드/백그라운드 클릭 트래킹(trackPushClickInternalEvent), 사용자 ID/프로퍼티 설정(setExternalUserId/setUserProperties), In-App 이벤트 리스너 추가/해제, SDK 래퍼 타입/버전 지정(setSdkType/setSdkVersion), 스케줄 푸시 테스트(schedulePushNotification) 등을 제공.
- [헬퍼](https://raw.githubusercontent.com/team-michael/notifly-ios-sdk/refs/heads/main/Sources/Notifly/notifly-ios-sdk/notifly-ios-sdk/SourceCodes/Notifly/Notifly+Helper.swift): 내부 유틸 및 공통 보조 로직. 파라미터 유효성 검사(예: projectId 정규식 확인), SDK 래퍼 버전/타입 접근자, 공용 유틸리티(스레드-세이프 호출 래핑 등)를 제공.

## 인프라스트럭처 - API
- [API 클라이언트](https://raw.githubusercontent.com/team-michael/notifly-ios-sdk/refs/heads/main/Sources/Notifly/notifly-ios-sdk/notifly-ios-sdk/SourceCodes/Infrastructures/API/NotiflyAPI.swift): 백엔드 HTTP 래퍼. 권한 부여(authorizeSession), 이벤트 업로드(trackEvent/재시도)와 사용자 상태 동기화(requestSyncState) 제공. 토큰 만료 메시지 감지 시 자동 갱신/재시도, Combine 기반 퍼블리셔 반환.
- [요청 모델/빌더](https://raw.githubusercontent.com/team-michael/notifly-ios-sdk/refs/heads/main/Sources/Notifly/notifly-ios-sdk/notifly-ios-sdk/SourceCodes/Infrastructures/API/NotiflyAPI+Request.swift): 요청 빌더/응답 디코더. 기본 헤더(X-Notifly-SDK-Version/Wrapper), Bearer 토큰 구성, JSON 인코딩, URLSession 퍼블리셔 래핑을 포함.

## 인프라스트럭처 - 인증
- [Auth](https://raw.githubusercontent.com/team-michael/notifly-ios-sdk/refs/heads/main/Sources/Notifly/notifly-ios-sdk/notifly-ios-sdk/SourceCodes/Infrastructures/Auth/Auth.swift): 사용자명/비밀번호 기반 세션 권한 부여 및 갱신, Authorization 헤더 발급/보관, 토큰 만료 시 재발급 흐름을 담당. NotiflyAPI 요청 경로에서 authorizationPub와 연계되어 자동으로 인증 토큰을 주입.

# Platform: React-native

## 코어 (TypeScript)
- [JS 공개 API (index)](https://raw.githubusercontent.com/team-michael/notifly-react-native-sdk/refs/heads/main/src/index.tsx): React Native에서 사용하는 공개 인터페이스. initialize/setUserId/getNotiflyUserId/setUserProperties/setEmail/setPhoneNumber/setTimezone/trackEvent/disableInAppMessage/addNotificationClickListener 등을 제공. iOS에서는 객체 파라미터(JSON.stringify) 변환을 적용, Android 전용 setLogLevel 노출. TurboModule 사용 여부(__turboModuleProxy) 확인 후 네이티브 모듈을 강제 로드하거나 링크 오류를 안내.
- [TurboModule 스펙](https://raw.githubusercontent.com/team-michael/notifly-react-native-sdk/refs/heads/main/src/NativeNotiflyReactNativeSdk.ts): 네이티브 브리지 인터페이스 정의. initialize, setUserId, setUserProperties, setEmail, setPhoneNumber, setTimezone, getNotiflyUserId, trackEvent, setLogLevel(Android 전용), disableInAppMessage, addNotificationClickListener 메서드를 약속하며 TurboModuleRegistry를 통해 강제 조회.
- [이벤트 상수](https://raw.githubusercontent.com/team-michael/notifly-react-native-sdk/refs/heads/main/src/constants.ts): 이벤트 이름 상수 정의. 예: Notifly#NotificationClicked.
- [이벤트 리시버](https://raw.githubusercontent.com/team-michael/notifly-react-native-sdk/refs/heads/main/src/events/EventReceiver.ts): NativeEventEmitter로 네이티브 이벤트를 구독/전달. 이벤트별 리스너 배열을 관리하고, NotificationClicked 이벤트의 rawPayload(JSON 문자열)를 객체로 안전 파싱하여 전달(실패 시 원본 반환 및 로그).
- [타입 (index)](https://raw.githubusercontent.com/team-michael/notifly-react-native-sdk/refs/heads/main/src/types/index.ts): InitializeParams(projectId/username/password), UserProperties/EventProperties 딕셔너리 타입 정의 및 notification 타입 재노출.
- [타입 (notification)](https://raw.githubusercontent.com/team-michael/notifly-react-native-sdk/refs/heads/main/src/types/notification.ts): OSNotification/OSNotificationClickEvent/OSNotificationImportance 정의. 캠페인 ID, 제목/본문, 중요도(Android), URL/이미지, 보낸 시각/TTL, 커스텀 데이터, 원본 페이로드(rawPayload) 등 필드 포함.

## iOS 네이티브 브리지
- [Obj‑C 브리지 선언](https://raw.githubusercontent.com/team-michael/notifly-react-native-sdk/refs/heads/main/ios/NotiflySdk.m): RCT_EXTERN_MODULE/_METHOD를 통해 Swift 구현을 React Native에 노출. initialize/setUserId/getNotiflyUserId/setUserProperties/trackEvent/disableInAppMessage/registerFCMToken 등을 Promise 기반으로 제공.
- [Swift 브리지 구현](https://raw.githubusercontent.com/team-michael/notifly-react-native-sdk/refs/heads/main/ios/NotiflySdk.swift): iOS Notifly SDK(notifly_sdk)를 호출하는 실제 구현. SDK 래퍼 타입/버전 설정, initialize, userId/프로퍼티 설정, 이메일/전화/타임존 설정, 이벤트 트래킹(JSON 파싱 포함), In‑App 비활성화, FCM 토큰 등록 등 제공.
- [브리징 헤더](https://raw.githubusercontent.com/team-michael/notifly-react-native-sdk/refs/heads/main/ios/NotiflySdk-Bridging-Header.h): Swift–ObjC 브리징을 위한 헤더 정의.

## Android 네이티브 브리지
- [RN 모듈 구현](https://raw.githubusercontent.com/team-michael/notifly-react-native-sdk/refs/heads/main/android/src/main/java/tech/notifly/rn/NotiflySdkModule.kt): ReactMethod로 네이티브 SDK 호출. SDK 래퍼 타입/버전 설정, initialize, userId/프로퍼티 설정, 이메일/전화/타임존, 이벤트 트래킹, 로그 레벨 설정(Android), In‑App 비활성화 및 알림 클릭 리스너(이벤트 전송 포함). 코루틴으로 비동기 처리/스레드 전환, DeviceEventEmitter로 JS 이벤트 송신.
- [유틸(이벤트 변환)](https://raw.githubusercontent.com/team-michael/notifly-react-native-sdk/refs/heads/main/android/src/main/java/tech/notifly/rn/NotiflySdkUtils.kt): 네이티브 객체를 React Native WritableMap으로 변환. NotificationClickEvent/Notification을 HashMap으로 매핑하고 각 필드를 JS 친화적 형태로 변환.
- [패키지 등록](https://raw.githubusercontent.com/team-michael/notifly-react-native-sdk/refs/heads/main/android/src/main/java/tech/notifly/rn/NotiflySdkPackage.kt): RN 패키지에 네이티브 모듈을 등록하여 자동 링크/수동 링크 시 인식.
- [스펙(new architecture)](https://raw.githubusercontent.com/team-michael/notifly-react-native-sdk/refs/heads/main/android/src/newarch/NotiflySdkSpec.kt): NewArch 템플릿용 스펙 정의.
- [스펙(old architecture)](https://raw.githubusercontent.com/team-michael/notifly-react-native-sdk/refs/heads/main/android/src/oldarch/NotiflySdkSpec.kt): OldArch 템플릿용 스펙 정의.
- [AndroidManifest](https://raw.githubusercontent.com/team-michael/notifly-react-native-sdk/refs/heads/main/android/src/main/AndroidManifest.xml): 패키지/퍼미션/프로바이더 등 Android 모듈 매니페스트 정의.
- [Gradle 빌드 스크립트](https://raw.githubusercontent.com/team-michael/notifly-react-native-sdk/refs/heads/main/android/build.gradle): Android 모듈 빌드/퍼블리시 설정.

# Platform: Google Tag Manager (GTM) Template

## 템플릿
- [GTM 템플릿 (영문)](https://raw.githubusercontent.com/team-michael/notifly-gtm-template/refs/heads/main/template.tpl): Notifly용 GTM 커스텀 템플릿(영문). 템플릿 필드/변수/권한 정의와 샌드박스 JS 구현을 포함. 이벤트/사용자 속성 전송 및 엔드포인트 호출 로직을 제공.
- [GTM 템플릿 (국문)](https://raw.githubusercontent.com/team-michael/notifly-gtm-template/refs/heads/main/template.ko.tpl): Notifly용 GTM 커스텀 템플릿(국문). 동일 기능을 한국어 라벨/도움말로 제공.
- [GTM 템플릿 (Cafe24 커스텀 이벤트)](https://raw.githubusercontent.com/team-michael/notifly-gtm-template/refs/heads/main/template-cafe24-custom-event.tpl): Cafe24 쇼핑몰에서 발생하는 커스텀 이벤트를 GTM을 통해 Notifly로 전달하기 위한 템플릿. 이벤트 매핑/필드 변환을 지원.
